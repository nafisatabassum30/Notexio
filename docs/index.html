Notexio Text Editor

A lightweight, customizable editor built in Python + Tkinter to demonstrate OS-project fundamentals (file I/O, event-driven GUI, persistence) with practical features like recovery, recent files, themes, and PDF export.

STAR report — challenges faced

1) Preventing data loss (unsaved changes + recovery)

  • Situation: Users can lose edits by closing/opening another file unexpectedly.
  • Task: Add safety checks and a lightweight recovery system.
  • Action:
    • Unsaved-change guard with Yes/No/Cancel save flow.
    • Recovery snapshots saved to recovery/ with timestamps; keep newest only.
    • Startup scan + restore prompt for newest recovery file.
  • Result: Fewer data-loss scenarios; predictable safety behavior for users.

2) Keeping code maintainable as features grew

  • Situation: Feature growth can turn a Tkinter app into a “god file”.
  • Task: Keep responsibilities separated and integration straightforward.
  • Action:
    • Split functionality into focused classes/modules (file ops, safety, UI, settings, etc.).
    • Centralize wiring in main.py (composition root).
    • Standardize UI updates (title/status/line numbers) to reduce duplicated state logic.
  • Result: Easier debugging and faster iteration; new features don’t ripple across the codebase.

3) Cross-platform extras (PDF export + printing + drag/drop)

  • Situation: Printing and drag/drop are OS-specific and easy to break.
  • Task: Deliver these features while preserving portability.
  • Action:
    • PDF export using ReportLab as an optional dependency.
    • Printing: Windows via optional pywin32; Linux/macOS via lp/lpr fallback.
    • Drag/drop: optional tkinterdnd2 on Windows; graceful fallback elsewhere.
  • Result: Feature-rich experience without OS lock-in; clear fallbacks and user messaging.
