STAR challenges report

STAR #1 — Preventing data loss (unsaved changes, recovery, auto-save)

  • Situation: Closing or crashing can instantly lose work; state can desync.
  • Task: Add a safe save/exit flow + recovery snapshots + restore on startup.
  • Action: tracked modified state (<<Modified>>), reused one save/exit gate (check_unsaved_changes), created bounded recovery snapshots, and prompted restore.
  • Result: Users get warned before losing work and can recover after unexpected exits.

STAR #2 — Responsive UI updates (status bar + optional line numbers)

  • Situation: Updating counts/position on every keystroke can cause lag.
  • Task: Provide Notepad-style live status readouts without slowing typing.
  • Action: centralized updates in update_status_bar, used after(...) scheduling, and made line numbers opt-in with guarded updates + scroll sync.
  • Result: Real-time feedback stays smooth; line numbers only cost when enabled.

STAR #3 — Cross-platform features with optional dependencies

  • Situation: Printing/PDF/drag & drop differ by OS and often need extra packages.
  • Task: Keep the core editor stable everywhere and enable extras when possible.
  • Action: platform detection + try/except ImportError, OS-specific print paths, and friendly fallback messages.
  • Result: The app runs everywhere; optional integrations activate only when supported.

2–5 minute demo script

  • 0:20–1:10: Create file → type → show * modified indicator → save.
  • 1:10–1:50: Attempt close with unsaved changes → show prompt; mention recovery snapshots.
  • 1:50–2:40: Find/Replace or Go To Line; toggle line numbers; show Open Recent updates.
  • 2:40–3:10: Wrap up: modular managers + graceful optional deps + data-loss prevention.
